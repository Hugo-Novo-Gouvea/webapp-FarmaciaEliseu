Produtos

Regras da Tela

- Tela que controla o CRUD de produtos
- Tabela que acessa o banco definido em configurações e a partir do endpoint da api recebe os valores de dbo.produtos onde ID diferente de 1 e deletado é false
- Tabela apresenta os itens "codigoBarras", "descricao", "precoCompra", "precoVenda" e "generico" da tabela
- o header da tabela é mantido enquanto o scroll é usado para melhor orientação do cliente
- Tabela tem paginação para evitar carregar vários itens e deixar o sistema pesado
- abaixo da paginação a esquerda é apresentado o numero de itens atuais e o numero máximo de itens
- abaixo da paginação a direita é apresentado os botões e avançar e voltar para navegar na paginação, ambos com seus emojis
- Acima da tabela tem o botão Menu, select box de coluna e input de pesquisa
- o Botão menu com seu emoji retorna para index.html
- select box apresenta qual a coluna que será usada de guia para pesquisas, impedindo que retorne valores referentes a outras colunas, as colunas permitidas são "codigoBarras" e "Descricao", mesmo "precoCompra", "precoVenda" e "generico" estando na tabela eles ficam de fora da pesquisa
- input de pesquisa recebe um valor a ser pesquisado na tabela e retornado na hora, esse valor podendo estar presente em qualquer parte do item referente a coluna (exemplo: caso eu pesquisa Ado, ele pode retornar a posição ado em qualquer parte do nome)
- Acima da área de pesquisa ficam os 4 botões do CRUD na ordem visualizar, Editar, Novo, Excluir com seus emojis
- Apenas o botão novo fica habilitado caso nenhum item da coluna seja selecionado
- ao selecionar ele habilita todos os botões

Visualizar
- retorna um modal com "codigoBarras", "codigoProduto", "descricao", "unidadeMedida", "precoCompra", "precoVenda", "localizacao", "laboratorio", "principio", "generico", "dataCadastro" e "dataUltimoRegistro" preenchidos e sem a capacidade de ser alterados

Editar
- retorna um modal com "codigoBarras", "codigoProduto", "descricao", "unidadeMedida", "precoCompra", "precoVenda", "localizacao", "laboratorio", "principio" e "generico" preenchidos e com a capacidade de ser alterados
- regras de cada campo:

	Codigo de Barras: fornece valor para o campo "codigoBarras" no dbo.produtos
	- mascara apenas numeros
	- Não pode ser nulo

	codigoProduto: fornece valor para o campo "codigoProduto" no dbo.produtos
	- mesmo recebendo letras minusculas deve ser salvo no banco com todas as letras maiusculas
	- pode ser nulo
	- caso seja nulo o banco recebe o valor "000000000"

	Descrição: fornece valor para o campo "descricao" no dbo.produtos
	- mesmo recebendo letras minusculas deve ser salvo no banco com todas as letras maiusculas
	- Não pode ser nulo

	Unidade de Medida: fornece valor para o campo "unidadeMedida" no dbo.produtos
	- mascara apenas numeros
	- pode ser nulo
	- caso seja nulo o banco recebe o valor 0

	Preço de Compra: fornece valor para o campo "precoCompra" no dbo.produtos
	- mascara apenas numeros
	- Não pode ser nulo
	- mostrar o valor sempre com dois zeros depois da virgula

	Preço de Venda: fornece valor para o campo "precoVenda" no dbo.produtos
	- mascara apenas numeros
	- Não pode ser nulo
	- mostrar o valor sempre com dois zeros depois da virgula

	Localização: fornece valor para o campo "localizacao" no dbo.produtos
	- mesmo recebendo letras minusculas deve ser salvo no banco com todas as letras maiusculas
	- pode ser nulo
	- caso seja nulo o banco recebe o valor "NAO INFORMADO"

	Laboratorio: fornece valor para o campo "laboratorio" no dbo.produtos
	- mesmo recebendo letras minusculas deve ser salvo no banco com todas as letras maiusculas
	- pode ser nulo
	- caso seja nulo o banco recebe o valor "NAO INFORMADO"

	principio: fornece valor para o campo "principio" no dbo.produtos
	- mesmo recebendo letras minusculas deve ser salvo no banco com todas as letras maiusculas
	- pode ser nulo
	- caso seja nulo o banco recebe o valor "NAO INFORMADO"

	generico: fornece valor para o campo "generico" no dbo.produtos
	- campo select box com valores "Selecione um valor", "SIM", "NAO", permitindo registro no banco apenas se valor diferente de "Selecione um valor"
	- não pode ser nulo

- ao ser feito o registro no banco é atualizado o campo "dataUltimoRegistro" com a data e horário atual
- botão cancelar para cancelar a operação
- botão salvar para realizar a operação
