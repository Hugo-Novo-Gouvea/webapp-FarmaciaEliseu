- Linha com as opções de venda em campos Select (Campos: TIPO DE VENDA, VENDEDOR, TIPO DE CLIENTE, TIPO DE PRODUTO, TIPO DE DESCONTO)

    - TIPO DE VENDA
        Selecione o tipo,
        Marcar,
        Dinheiro
    Define se o deletado do movimento vai ser false ou true, sendo marcar false e dinheiro true

    VENDEDOR
        - Mostra "Selecione um Vendedor"
        - Retorna todos os items em funcionarios e deixa o usuario selecionar um

    - TIPO DE CLIENTE
        Selecione o tipo,
        Registrado,
        Avulso
    Caso Registrado: Cria uma nova linha com um campo input onde pode se procurar o cliente e abaixo do campo vai retornando valores semenlhantes aos digitados para auxiliar o usuario e ao lado o total que assim que escolhido o cliente mostra o fiado atual do cliente sendo a soma de todos os precoTotalAtual dos movimentos desse cliente onde deletado = false
    Caso Avulso: Campo input para que o usuario digite o nome do cliente sem busca pois é um cliente fora do banco

    - TIPO DE PRODUTO
        Selecione o tipo,
        Registrado,
        Outras Vendas
    Caso Registrado: 
        - Cria uma nova linha com um campo input (Codigo de Barras/Descrição) onde o usuario pode informar a descrição ou codigo de barras de um produto e ao apertar enter ele move o foco para o campo quantidade que está ao lado datUltimoRegistro
        - Se for digitado um valor em descrição e apertado Enter e não bater com nenhum codigo de barras existente deve ser retornado um modal com todos os itens com descrição(apenas descrição) semenlhante ao valor digitado e o usuario poderá selecionar um e apertar em confirmar ou cancelar no botão abaixo da tabelas de itens referente e então esse que será adicionado no campo Codigo de Barras/Descrição
        - Campo quantidade é um campo do tipo number que só aceita de 1 para cima e ao apertar Enter muda o foco para o campo desconto que fica logo ao lado direito
        - campo desconto começa em 0 tipo number e recebe valores quebrados(se tiver outras forma de tratar q n seja number n tem problema) e ao apertar Enter registra o item na tabela
        - Botão Adicionar caso o usuario não queira apertar o Enter no desconto ele adicionar(mesmo estilo de Realizar Venda) na tabela
    Caso Avulso: Campo input para que o usuario digite o nome do cliente sem busca pois é um cliente fora do banco

    - TIPO DE DESCONTO
        Porcentagem,
        Fixo
    Caso Porcentagem: o desconto digitado do item vai ser aplicado em porcentagem exemplo: se dititado 18 então é aplicado 18% de desconto
    Caso Fixo: o desconto digitado do item vai ser aplicado em valor fixo exemplo: se dititado 18 então é aplicado 18 reais de desconto

Regras do Movimento

- Tela que controla as vendas do cliente

- Tabela

- Realizar venda coloca em movimentos as informações referente aos produtos sendo

movimentoID: PK
produtosID: ID do produto ou se for avulso colocar 1
produtosDescricao: descricao do produto escolhido no select box ou digitado no input de avulsos
produtosCodigoProduto: codigoProduto do produto escolhido ou "000000000" se avulso
clientesID: ID do cliente escolhido na select box ou 1 se cliente avulso
clientesNome: nome do cliente escolhido na select box ou digitado no input se avulso
funcionariosID: ID do funcionario escolhido no select box funcionarios
funcionariosNome: Nome do funcionario escolhido no select box funcionarios
quantidade: quantidade do produto digitado tanto em avulso quando registrado
precoUnitarioDiaVenda: registro do precoVenda do produto feito no dia do registro
precoTotalDiaVenda: registro do precoVenda x quantidade do produto feito no dia do registro
precoUnitarioAtual: registro do precoVenda do produto feito no dia do registro (Sim é semelhante ao DiaVenda)
precoTotalAtual: registro do precoVenda x quantidade do produto feito no dia do registro (Sim é semelhante ao DiaVenda)
valorPago: se escolhido Dinheiro então recebe precoVenda x quantidad e se escolhido marcar recebe 0
desconto: recebe o desconto digitado
DataVenda: data atual do dia que foi criado o registro
DataPagamento: Null se Marcado, Data atual do dia que foi criado se Dinheiro
codigoMovimento: Esse é meio complicado, ele serve pra poder agrupar os movimentos em grupos de vendas, se eu vendi X,Y,Z remedio na mesma venda pra um cliente eles tem que ter o mesmo codigoMovimento e nunca pode repetir
dataCadastro: Controlado pelo sistema, data atual e horario em que a venda foi feita
datUltimoRegistro: Controlado pelo sistema, data atual e horario em que a venda foi feita
deletado: Se escolher Marcar o deletado do movimento = false e se escolher dinheiro o deletado do movimento = true
